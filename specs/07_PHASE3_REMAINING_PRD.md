## 1. 概要

### 1.1 目的

案件のムーブメント進捗をタイムライン形式（ガントチャート）で可視化し、案件全体のスケジュール感と各ステップの所要期間を一目で把握できるようにする。

### 1.2 スコープ

| 機能 | 説明 |
|------|------|
| **ムーブメント一覧のガントチャート表示** | 既存の `/movements` ページにマトリクス表示との切替タブを追加 |
| **案件詳細のガントチャート** | 案件詳細のムーブメントタブ内にガントチャートを追加 |
| **時間軸の切替** | 日/週/月表示の切替 |
| **ステップ別バー表示** | 各ステップの開始〜完了を色付きバーで表示 |

### 1.3 スコープ外

| 機能 | 理由 |
|------|------|
| ムーブメントログ（変更履歴） | 不要と判断 |
| 営業ステータス遷移制御（優先度ベース） | 不要と判断 |
| ドラッグ&ドロップによるスケジュール変更 | 複雑度が高い。バーのクリックで編集モーダルを開く方式で代替 |

### 1.4 Phase 3 で削除された機能

以下はPRD原案に含まれていたが、実務上不要と判断し削除した:

- **ムーブメントログ**: `ProjectMovement.updatedAt` / `updatedBy` で最終更新は追跡可能。全変更履歴の記録は運用上不要
- **営業ステータス遷移制御（優先度ベース）**: `BusinessStatusDefinition.statusPriority` フィールドは存在するが、遷移制御のバリデーションは実装しない。運用ルールで対応

## 2. ガントチャートの設計

### 2.1 データソース

ガントチャートのバー描画に使用するデータ:

| データ | ソース | 用途 |
|--------|--------|------|
| ステップ開始日 | `ProjectMovement.movementStartedAt` | バーの開始位置 |
| ステップ完了日 | `ProjectMovement.movementCompletedAt` | バーの終了位置 |
| ステップ状態 | `ProjectMovement.movementStatus` | バーの色と表現 |
| ステップ名 | `MovementTemplate.stepName` | バーのラベル |
| 受注予定月 | `Project.projectExpectedCloseMonth` | 目標線の描画 |

**バーの描画ルール:**

| ステータス | 表現 |
|-----------|------|
| `pending` | 描画しない（日付がないため。行には「未着手」テキストを表示） |
| `started` | 青バー（開始日 〜 現在日） |
| `completed` | 緑バー（開始日 〜 完了日） |
| `skipped` | 灰色の点線バー |

### 2.2 配置先と表示モード

#### A. ムーブメント一覧ページ（`/movements`）

既存のマトリクス表示に加え、ガントチャート表示を切替可能。

```
[マトリクス表示] [ガントチャート表示]    ← トグルボタン
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                  2025/01  2025/02  2025/03  2025/04
ABC商事           ████     ████
MG-0001                    ████     ████
                                    ████     ████  |← 受注予定月
DEF工業           ████████████              ▼今日
MG-0002                    ████████████████
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- **縦軸**: 案件（顧客名 + 案件番号）
- **横軸**: 時間（日/週/月で切替）
- **バー**: 各ステップを色分けして表示
- **目標線**: 受注予定月にオレンジの点線
- **今日線**: 赤の縦線 + 「今日」ラベル
- 既存のフィルター（営業ステータスフィルター）はそのまま適用

#### B. 案件詳細ページ（ムーブメントタブ内）

既存のマトリクス表の下にガントチャートを追加。単一案件の全ステップを時系列表示する。

```
ステップ1: 契約前確認    ████████████
ステップ2: 契約締結              ████████
ステップ3: 機器発注                      ████████████
ステップ4: 設置工事                              ████ ← 進行中
ステップ5: 検収                                       (未着手)
                        1月    2月    3月    4月    5月
```

### 2.3 時間軸の切替

| 表示 | 横軸 | 1列の幅 | ユースケース |
|------|------|---------|-------------|
| 日表示 | 日 | 36px | 直近のタスク進捗の細かい確認。週末は赤背景 |
| 週表示 | 週（月曜始まり） | 80px | 直近のタスク進捗の詳細確認 |
| 月表示 | 月 | 120px | 中長期の全体把握。受注予定月との比較 |

- デフォルトは月表示
- 切替はボタングループ（「日」「週」「月」）

### 2.4 タイムライン範囲の算出

タイムライン範囲は以下の要素を全て含むように自動算出:

1. 全バーの開始日・終了日
2. **受注予定月の月末日**（目標線が必ず範囲内に入る）
3. **今日の日付**（今日線が必ず範囲内に入る）
4. 前後2週間の余白

### 2.5 インタラクション

| 操作 | 動作 |
|------|------|
| バーのクリック | ムーブメント編集モーダルを開く（既存の `MovementEditModal` を再利用） |
| バーのホバー | ツールチップ表示（ステップ名、日付範囲、ステータス） |
| 案件名クリック（一覧） | 案件詳細ページへ遷移 |
| 横スクロール | タイムラインをスクロール（案件名列は左側に固定） |
| 初期表示 | 今日付近に自動スクロール |

### 2.6 凡例

チャート上部に凡例を表示:

```
■ 完了  ■ 進行中  □ スキップ  ┄ 受注予定月
```

## 3. API

### 3.1 既存APIの活用

ガントチャートに必要なデータは既存APIで取得可能。新規APIは不要。

| 用途 | API | 備考 |
|------|-----|------|
| 一覧ガントチャート | `GET /api/v1/projects/movements?businessId=X` | 既存。全案件のムーブメント + テンプレート情報を返す |
| 案件詳細ガントチャート | `GET /api/v1/projects/:id/movements` | 既存。単一案件のムーブメント一覧 |

レスポンスにはすでに `movementStartedAt`、`movementCompletedAt`、`movementStatus`、テンプレート情報が含まれている。

## 4. 技術選定

### 4.1 実装方式

**フルスクラッチ実装**（外部ライブラリ不使用）。

| 項目 | 内容 |
|------|------|
| 描画方式 | HTML/CSS（Tailwind CSS）ベースのDiv要素でバーを描画 |
| 位置計算 | パーセントベースの絶対位置配置 |
| スクロール | ネイティブの `overflow-x-auto` |
| ツールチップ | `position: fixed` のホバーポップアップ |

**当初 Frappe Gantt を計画していたが変更した理由:**

| 理由 | 詳細 |
|------|------|
| デザイン統一 | Tailwind CSS + shadcn/ui のデザインシステムと完全に統合 |
| 柔軟性 | 受注予定月マーカー、ステータス色分け、ラベル列固定などを自由にカスタマイズ |
| 軽量性 | 外部ライブラリの依存なし |
| SSR対応 | `'use client'` ディレクティブのみで動作。dynamic import 不要 |

### 4.2 コンポーネント構成

```
src/components/features/project/
├── gantt-chart.tsx              # メインコンポーネント（描画・インタラクション）
├── gantt-chart-utils.ts         # データ変換・時間軸計算・ユーティリティ
├── project-movements-tab.tsx    # 既存（マトリクス表 + ガントチャートを統合）
└── movement-edit-modal.tsx      # 既存（再利用）

src/app/(auth)/movements/
├── _client.tsx                  # 既存（マトリクス/ガントチャートの表示切替を追加）
└── page.tsx                     # 既存
```

## 5. レスポンシブ対応

- デスクトップファースト（最低1280px幅）
- 案件名列は固定（左側 200px）、タイムライン部分は横スクロール
- 日表示: 1列 = 36px
- 週表示: 1列 = 80px
- 月表示: 1列 = 120px
- 表示範囲は全バー + 受注予定月 + 今日 + 前後2週間で自動算出
