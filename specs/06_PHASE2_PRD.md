## 1. 概要

### 1.1 目的

案件（Project）のCRUD管理を実装し、顧客・代理店と案件を紐付けて営業活動を管理できるようにする。
併せて、事業マスタの詳細画面に営業ステータス定義・ムーブメントテンプレート・案件フィールド定義の管理タブを追加し、事業ごとの設定をUI上で行えるようにする。

### 1.2 Phase 2 のスコープ

| 機能 | 説明 |
|------|------|
| **案件CRUD** | 共通固定項目 + 事業固有フィールド（動的フォーム）による案件管理 |
| **案件番号自動採番** | `{事業プレフィックス}-{連番}` 形式（例: `MG-0001`） |
| **案件一覧** | フィルター・検索・ソート・インライン編集・ページネーション |
| **事業セレクター連動** | ヘッダーの事業セレクターで選択した事業の案件のみ一覧表示 |
| **事業詳細の管理タブ** | 営業ステータス定義 / ムーブメントテンプレート / 案件フィールド定義 のCRUD管理 |
| **動的フィールド** | `businessConfig.projectFields` からフォーム・一覧・詳細を動的に生成 |
| **ムーブメント自動生成** | 案件作成時にテンプレートから `pending` レコードを自動生成（UIはPhase 3） |
| **CSV** | 案件のインポート / エクスポート（共通項目 + 事業固有フィールド対応） |
| **関連案件タブ** | 顧客詳細・代理店詳細に「関連案件」タブを追加 |

### 1.3 Phase 2 のスコープ外

| 機能 | 実装Phase |
|------|-----------|
| ムーブメント操作UI・ステータス連動 | Phase 3（実装済み） |
| ガントチャート | Phase 3 |
| ダッシュボード・予実管理 | Phase 4 |
| ファイル管理（案件添付） | Phase 5 |

## 2. 案件の項目設計

### 2.1 共通固定項目

全事業の案件に必ず存在する項目。`projects` テーブルのカラムとして定義する。
事業マスタのフィールド定義では削除できない。

| 項目 | DB カラム | 型 | 必須/任意 | 備考 |
|------|-----------|-----|-----------|------|
| 案件番号 | `project_no` | VARCHAR(30) | 自動採番 | 読取専用。`{事業プレフィックス}-{連番}` |
| 顧客 | `customer_id` | INTEGER FK | **必須** | 顧客マスタから選択 |
| 代理店 | `partner_id` | INTEGER FK | 任意 | 代理店マスタから選択。直販案件はNULL |
| 営業ステータス | `project_sales_status` | VARCHAR(50) | **必須** | 事業のステータス定義から選択 |
| 受注予定月 | `project_expected_close_month` | VARCHAR(7) | 任意 | `YYYY-MM` 形式 |
| 担当者名 | `project_assigned_user_name` | VARCHAR(100) | 任意 | 自由記入テキスト |
| 担当ユーザー | `project_assigned_user_id` | INTEGER FK | 任意 | アクセス制御用。ユーザーマスタから選択 |
| 備考 | `project_notes` | TEXT | 任意 | |

### 2.2 事業固有フィールド

事業マスタの `businessConfig.projectFields` で定義し、`projects.project_custom_data`（JSONB）に格納する。
事業ごとに案件名・金額・日付・台数など自由に項目を追加できる。

**対応する型:**

| 型 | 説明 | 例 |
|----|------|----|
| `text` | テキスト入力 | 案件名、担当者名 |
| `textarea` | 複数行テキスト | 詳細メモ |
| `number` | 数値入力 | 金額、台数 |
| `date` | 日付選択 | 受注日、納品日 |
| `month` | 月選択 | 予定月 |
| `select` | ドロップダウン選択 | ステータス、区分 |
| `checkbox` | チェックボックス | フラグ項目 |

### 2.3 案件番号の採番ルール

- 形式: `{事業プレフィックス}-{4桁連番}` （例: `MG-0001`, `SA-0001`）
- 事業プレフィックスは `businesses.businessProjectPrefix` から取得
- 同一事業内で最新の案件番号 +1 の単純採番
- 自動生成、読取専用（手動変更不可）

## 3. 事業マスタ管理タブの拡張

Phase 2 で事業詳細画面に以下の3つのCRUD管理タブを追加する。

### 3.1 営業ステータス定義タブ

事業ごとの営業ステータスをインラインCRUDで管理する。

| 管理項目 | 説明 |
|----------|------|
| ステータスコード | 一意識別子。作成後は変更不可 |
| 表示ラベル | 画面表示名（例: "1.購入済み"） |
| 優先度 | 数値（Phase 3の遷移制御で使用） |
| 表示色 | カラーコード |
| 最終ステータス | フラグ（事業内で1つのみ） |
| 失注ステータス | フラグ（事業内で1つのみ） |
| 表示順 | ドラッグ＆ドロップで並び替え |

### 3.2 ムーブメントテンプレートタブ

事業ごとのムーブメントステップをインラインCRUDで管理する。

| 管理項目 | 説明 |
|----------|------|
| ステップ番号 | 自動採番、並び替え時に再計算 |
| ステップコード | 一意識別子。作成後は変更不可 |
| ステップ名 | 画面表示名 |
| 説明 | ステップの詳細説明 |
| 営業ステータス連動 | フラグ（Phase 3で連動ロジック実装） |
| 連動ステータスコード | 連動する営業ステータス |

### 3.3 案件フィールド定義タブ

事業固有の案件フィールドを管理する。ここで定義したフィールドが案件フォーム・一覧・詳細に動的に反映される。

| 管理項目 | 説明 |
|----------|------|
| フィールドキー | 一意識別子（`project_custom_data` のJSONキー） |
| 表示ラベル | 画面表示名 |
| 型 | text / textarea / number / date / month / select / checkbox |
| 選択肢 | select型の場合の選択肢リスト |
| 必須フラグ | 入力必須かどうか |
| 表示順 | フォーム・一覧での表示順 |
| 説明 | フィールドの補足説明（プレースホルダー等） |

## 4. 案件一覧

### 4.1 表示列

**固定列（常に表示）:**
- 案件番号
- 顧客名
- 営業ステータス

**共通列:**
- 代理店名
- 受注予定月
- 担当者名
- 備考
- 作成日
- 更新日

**事業固有列:**
- `businessConfig.projectFields` で定義されたフィールドを動的に列として追加

### 4.2 検索・フィルター

| 機能 | 対象 |
|------|------|
| テキスト検索 | 案件番号、顧客名、代理店名、担当者名 |
| フィルター | 営業ステータス（multi-select）、担当者名（テキスト）、受注予定月（month-range） |
| 事業セレクター連動 | ヘッダーの事業セレクターで選択した事業でフィルタリング |

### 4.3 インライン編集

- 営業ステータス（select: 事業のステータス定義から）
- 受注予定月（month）
- 担当者名（text: 自由記入）
- 備考（textarea）
- 事業固有フィールド（型に応じたエディタ）

### 4.4 一括操作

| アクション | 説明 |
|------------|------|
| 一括削除 | 選択した案件の論理削除 |

## 5. 案件フォーム（新規作成・編集）

### 5.1 フォーム構成

```
セクション1: 基本情報
├── 顧客（検索付きセレクト、必須）
├── 代理店（検索付きセレクト、任意）
├── 営業ステータス（セレクト、必須。事業のステータス定義から）
├── 受注予定月（月選択）
├── 担当者名（テキスト、自由記入）
├── 担当ユーザー（セレクト、アクセス制御用。ユーザーマスタから）
└── 備考（テキストエリア）

セクション2: 事業固有項目（businessConfig.projectFields から動的生成）
├── [フィールド定義に基づく動的フォーム]
└── ...
```

### 5.2 バリデーション

- 共通項目: Zodスキーマで静的バリデーション
- 事業固有項目: `businessConfig.projectFields` の定義（型・必須フラグ）から動的にZodスキーマを生成

### 5.3 案件作成時の自動処理

1. 案件番号の自動採番
2. ムーブメントレコードの自動生成（テンプレートの全ステップ分、`pending` 状態で作成）

## 8. 事業セレクター連動

### 8.1 概要

ヘッダーの事業セレクターで選択した事業に基づき、案件一覧をフィルタリングする。

### 8.2 動作仕様

| セレクター値 | 動作 |
|-------------|------|
| 「すべて」 | 全事業の案件を表示（admin: 全事業、staff: アサイン済み事業） |
| 特定事業 | その事業の案件のみ表示 |

### 8.3 実装方針

- 事業セレクターの選択値をグローバルstate（Zustand）で管理
- 案件一覧APIに `businessId` パラメータを追加
- 案件新規作成時は選択中の事業をデフォルトにセット

## 9. CSV インポート/エクスポート

### 9.1 エクスポート

- 共通項目 + 事業固有フィールドを列として出力
- 事業セレクターで絞り込んだ状態でエクスポート
- 表示列のみ / 全列の選択式

### 9.2 インポート

- 3ステップモーダル（既存テンプレート流用）
- 顧客コード → `customer_id` の名寄せ
- 代理店コード → `partner_id` の名寄せ（空欄はNULL）
- 営業ステータスコード → バリデーション（事業のステータス定義に存在するか）
- 事業固有フィールドは `project_custom_data` に格納

## 10. 共通コンポーネント設計方針

Phase 2で新規作成・拡張する共通コンポーネント:

| コンポーネント | 種別 | 用途 |
|---------------|------|------|
| `DynamicFormSection` | 新規・共通 | `projectFields` 定義から動的にフォームフィールドを生成 |
| `DynamicColumns` | 新規・共通 | `projectFields` 定義から動的にテーブル列を生成 |
| `EntitySelectField` | 新規・共通 | 顧客・代理店・ユーザー等のエンティティ検索付き選択 |
| `SortableItemList` | 新規・共通 | ドラッグ＆ドロップ並び替え付きCRUDリスト（ステータス定義・テンプレート・フィールド定義で共用） |
| `BusinessScopeProvider` | 新規・共通 | 事業セレクター連動のグローバルstate管理 |
| `MonthPicker` | 新規・共通 | 月選択UI（`YYYY-MM` 形式） |
| `ColorPicker` | 新規・共通 | ステータス色選択UI |

**既存コンポーネントの拡張:**

| コンポーネント | 拡張内容 |
|---------------|----------|
| `EntityListTemplate` | 事業セレクター連動、動的列対応 |
| `EntityFormTemplate` | 動的フォームセクション対応 |
| `EntityDetailTemplate` | 動的フィールド表示対応 |
| 顧客詳細Config | 関連案件タブ追加 |
| 代理店詳細Config | 関連案件タブ追加 |

## 11. 実装順序

```
Step 1: Prismaスキーマ拡張
  └── Project / ProjectMovement モデル追加 + マイグレーション

Step 2: 事業マスタ管理タブ
  ├── 営業ステータス定義 CRUD API + タブUI
  ├── ムーブメントテンプレート CRUD API + タブUI
  └── 案件フィールド定義 CRUD API + タブUI
      └── SortableItemList 共通コンポーネント

Step 3: 共通コンポーネント
  ├── EntitySelectField（検索付きエンティティ選択）
  ├── DynamicFormSection / DynamicColumns
  ├── BusinessScopeProvider（事業セレクター連動）
  └── MonthPicker / ColorPicker

Step 4: 案件CRUD
  ├── 案件API（一覧・詳細・作成・更新・削除・復元）
  ├── 案件番号自動採番
  ├── ムーブメント自動生成
  ├── 案件Config（list / detail / form）
  └── 案件ページ（一覧・詳細・新規・編集）

Step 5: CSV + 一括操作
  ├── 案件CSV API
  └── 一括削除API

Step 6: 関連機能
  ├── 顧客詳細・代理店詳細に関連案件タブ追加
  ├── 事業セレクター連動実装
  └── シードデータ更新

Step 7: 動作検証
  └── 全機能の結合テスト
```
