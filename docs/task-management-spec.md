# タスク管理機能 仕様書

## 基本方針

| 項目 | 決定事項 |
|------|---------|
| 紐付け | 汎用タスク（案件・顧客・代理店に任意紐付け可 / なしも可） |
| 利用者 | 社内ユーザーのみ（admin / staff） |
| 表示形式 | カンバン + リスト + カレンダーの3ビュー切替 |
| 自動生成 | なし（手動作成のみ） |
| スコープ | マイタスク + グループボード（複数）構成 |
| サブタスク | 階層型サブタスク（2階層まで）+ チェックリスト 両方 |
| 優先度 | 緊急・高・中・低の4段階 |
| 担当者 | 複数担当者対応（アカウントユーザー + フリーテキスト外部担当者） |
| タグ | 個人タグ + 共通タグ（全ユーザー作成可）+ **チップ一覧選択** + サジェスト重複抑制 |
| 通知 | タスク毎に通知レベル・通知先を設定可能 + 期限通知Cron |
| 備考 | メモ（備考）欄あり（コメント機能は不採用） |
| 期限 | 期限なし（未設定）も選択可能 |
| 添付ファイル | S3アップロード + D&D + クリップボード貼り付け + 画像プレビュー |
| ダッシュボード | タスクウィジェット（サマリー + 期限間近 + 期限超過） |
| レスポンシブ | CSS-only（Tailwind sm:）+ scroll-snap カンバン + ボトムシートモーダル |

---

## データモデル

### Task（タスク）

```
Task
├── id                    : Int @id @default(autoincrement())
├── taskNo                : String @unique（自動採番: TASK-0001）
├── title                 : String（タスク名、最大200文字）
├── description           : String?（説明）
├── status                : String（'todo' | 'in_progress' | 'done' | 'on_hold'）
├── priority              : String（'urgent' | 'high' | 'medium' | 'low'）
├── dueDate               : DateTime?（期限、null = 期限なし）
├── taskUrl               : String?（関連URL、最大500文字）
├── createdById           → User（作成者）
├── boardId               → TaskBoard?（グループボード紐付け、null = マイタスク）
├── columnId              → TaskColumn?（カンバン列）
├── parentTaskId          → Task?（階層型サブタスク：自己参照、2階層まで）
├── children              : Task[]（子タスク一覧）
├── assignees             : TaskAssignee[]（複数担当者）
├── checklist             : Json [{ id, text, checked }]（手順チェック）
├── sortOrder             : Int（カンバン内の並び順）
├── relatedEntityType     : String?（'project' | 'customer' | 'partner'）
├── relatedEntityId       : Int?
├── notifyLevel           : String（'none' | 'in_app' | 'in_app_and_email'）デフォルト: 'in_app'
├── memo                  : String?（備考・メモ欄）
├── notifyTargets         : TaskNotifyTarget[]（通知先リスト）
├── tags                  : TaskTagOnTask[]（多対多）
├── attachments           : TaskAttachment[]（添付ファイル）
├── completedAt           : DateTime?
├── version               : Int @default(1)（楽観的ロック）
├── createdAt, updatedAt
```

※ `assigneeId` は廃止。複数担当者は `TaskAssignee` テーブルで管理。
※ `scope` / `businessId` は廃止。`boardId` でグループボード紐付けを管理。`TASK_SCOPE_OPTIONS` 削除。

### TaskAssignee（タスク担当者）

```
TaskAssignee
├── id                    : Int @id @default(autoincrement())
├── taskId                → Task（onDelete: Cascade）
├── userId                → User?（nullable: アカウント未登録の外部担当者はnull）
├── userName              : String（担当者名、必須。userId設定時はユーザー名を格納）
├── assignedAt            : DateTime @default(now())
├── @@unique([taskId, userId])  ※userId非nullの場合のみユニーク
```

- **アカウントユーザー**: `userId` + `userName`（ユーザー名を自動セット）
- **フリーテキスト担当者**: `userId = null` + `userName`（手入力の名前のみ）
- UI: ユーザー選択ドロップダウン + フリーテキスト入力の複合入力
- 表示: 先頭の担当者名 + 「+N」（複数人の場合）

### TaskAttachment（添付ファイル）

```
TaskAttachment
├── id                    : Int @id @default(autoincrement())
├── taskId                → Task（onDelete: Cascade）
├── fileName              : String（元のファイル名、最大255文字）
├── fileKey               : String（S3キー、最大500文字）
├── fileSize              : Int（バイト数）
├── mimeType              : String（image/png, application/pdf等、最大100文字）
├── uploadedById          → User（アップロード者）
├── createdAt
```

- アップロード: ファイル選択 / ドラッグ&ドロップ / Ctrl+V クリップボード貼り付け
- 制限: 1ファイル10MB上限、1タスク10件上限
- 画像ファイル: サムネイルプレビュー表示、クリックで拡大モーダル
- 削除: アップロード者本人 or admin

### TaskColumn（カンバン列）

```
TaskColumn
├── id                    : Int @id @default(autoincrement())
├── name                  : String（列名、最大100文字）
├── color                 : String?（表示色: hex, e.g. '#3b82f6'）
├── sortOrder             : Int @default(0)（表示順）
├── boardId               → TaskBoard?（ボード別列管理、null = マイタスク用）
├── createdById           → User（作成者）
├── tasks                 : Task[]
├── createdAt, updatedAt
```

- 初回アクセス時にデフォルト4列（未着手/進行中/保留/完了）を自動作成
- マイタスク / 各グループボードごとに独立した列セットを管理
- 列の追加・名前変更・色変更・削除・並び替えが可能
- 列削除時、所属タスクの `columnId` は `null` に設定

### TaskTag（タグ）

```
TaskTag
├── id                    : Int @id
├── name                  : String（タグ名、最大100文字）
├── color                 : String（表示色: hex, e.g. '#ef4444'）
├── scope                 : String（'shared' | 'personal'）
├── ownerId               → User（作成者）
├── @@unique([name, scope, ownerId])
```

### TaskBoard（グループボード）

```
TaskBoard
├── id                    : Int @id @default(autoincrement())
├── name                  : String（ボード名、最大100文字）
├── description           : String?（説明）
├── createdById           → User（作成者）
├── members               : TaskBoardMember[]
├── tasks                 : Task[]
├── createdAt, updatedAt
```

### TaskBoardMember（ボードメンバー）

```
TaskBoardMember
├── boardId               → TaskBoard
├── userId                → User
├── role                  : String（'owner' | 'member'）  ※将来拡張用
├── tabOrder              : Int @default(0)（ボードタブの表示順序）
├── joinedAt              : DateTime
├── @@id([boardId, userId])
```

### 中間テーブル

- TaskTagOnTask: taskId + tagId の複合主キー
- TaskNotifyTarget: taskId + userId の複合主キー

## 担当者の仕組み

### 複数担当者対応

| 機能 | 仕様 |
|------|------|
| データ構造 | `TaskAssignee` 多対多テーブル（旧 `assigneeId` は廃止） |
| アカウントユーザー | `userId` + `userName` で登録。ユーザー選択ドロップダウンから選択 |
| フリーテキスト担当者 | `userId = null` + `userName` のみ。外部関係者など未登録ユーザー用 |
| UI入力 | マルチセレクトユーザーピッカー + フリーテキスト入力欄 |
| 一覧表示 | 先頭の担当者名のみ表示、複数人の場合は「+N」バッジ |
| 詳細パネル | 全担当者を一覧表示、個別削除可能 |

## タグの仕組み

| | 共通タグ（shared） | 個人タグ（personal） |
|---|---|---|
| 作成 | 全ユーザーが作成可 | 全ユーザーが自分用に作成 |
| 可視範囲 | 全ユーザーに表示 | 作成者のみに表示 |
| タスクへの付与 | 誰でも付与可 | 作成者のみ付与可 |
| 編集・削除 | 作成者 + admin | 作成者のみ |

### 共通タグのチップ一覧選択
- タグ入力欄の**上**に共通タグをカラーチップで一覧表示
- クリックで即付与/解除（トグル動作）
- 入力欄は追加検索・個人タグ選択・新規作成用

### 重複抑制（サジェスト機能）
- タグ入力時に既存の共通タグ + 個人タグを候補表示
- 類似タグがある場合「似たタグがあります」とサジェスト
- 新規作成時にスコープ（共通 / 個人）と色を選択

## チェックリスト vs サブタスクの明確化

| | チェックリスト ☑ | サブタスク 📋 |
|---|---|---|
| アイコン | ☑ チェックボックス（オレンジ） | 📋 タスクアイコン（青） |
| ラベル | 「チェックリスト」 | 「サブタスク」 |
| 担当者 | なし | あり（複数設定可） |
| 期限 | なし | あり（個別設定可） |
| ステータス | ☑ / ☐ のみ | todo / in_progress / done / on_hold |
| 通知 | なし | 独自の通知設定を持てる |
| 進捗への影響 | しない | 親タスクの進捗率に影響 |
| 用途 | タスク内の作業手順メモ | 分業・委任が必要な作業単位 |

### 詳細パネルでの表示

```
┌────────────────────────────────┐
│ ☑ チェックリスト (2/3)    [+ 追加] │
│ ■■□ ──────────────────────     │
│ ☑ 原価確認済み                  │
│ ☑ 契約書テンプレート準備         │
│ ☐ 上長承認                     │
├────────────────────────────────┤
│ 📋 サブタスク (1/3完了)    [+ 追加] │
│ ■□□ ──────────────────────     │
│ ✅ ヒアリング  田中+1  3/20     │  ← クリックで子タスク詳細に遷移
│ 🔄 見積作成   佐藤     3/22     │
│ ⬜ レビュー   鈴木     3/25     │
└────────────────────────────────┘
```

## サブタスクの階層表示（リストビュー）

親タスク展開時、ツリー記号で階層構造を視覚化:

```
TASK-0001  A社 新規提案       進行中  🔴緊急  田中+1  3/25  [重要顧客]
┣ TASK-0002  ヒアリング実施   完了    —      田中    3/20
┣ TASK-0003  見積作成        進行中   —      佐藤    3/22
┗ TASK-0004  提案書レビュー   未着手   —      鈴木    3/25
```

- `┣` 中間の子タスク、`┗` 最後の子タスク
- 子タスク行はインデント + 薄い左ボーダーで視覚的に親子関係を表現
- 子タスクの優先度列は `—` 表示（親から継承、個別表示不要）

## 親→サブタスクのパネル遷移 + パンくず

### 遷移フロー

1. 親タスク詳細パネルで子タスク行をクリック
2. パネル内容が子タスク詳細に切り替わる（パネルは閉じない）
3. パネル上部にパンくずナビゲーション表示

### パンくず表示

```
┌──────────────────────────────────────────┐
│ TASK-0001 A社 新規提案 > TASK-0002 ヒアリング  × │
├──────────────────────────────────────────┤
│ ヒアリング実施                              │
│ ステータス: [完了 ▼]  優先度: [中 ▼]          │
│ ...                                       │
└──────────────────────────────────────────┘
```

- パンくずの親部分（`TASK-0001 A社 新規提案`）はクリック可能で親に戻る
- 子タスク詳細では「手順チェック」「メモ」等は表示するが「子タスク」セクションは非表示（2階層制限のため）

## 通知設定

タスク毎に通知レベルと通知先を設定可能。

| 設定 | 挙動 |
|------|------|
| `none` | 一切通知しない |
| `in_app` | アプリ内通知のみ（デフォルト） |
| `in_app_and_email` | アプリ内 + メール |

通知先（notifyTargets）はデフォルトで担当者が入り、任意のユーザーを追加可能。

### 期限通知Cron

- エンドポイント: `/api/v1/cron/task-notifications`（`CRON_SECRET` 認証）
- **期限1日前通知**: 翌日が期限のタスクに対して通知送信
- **期限超過通知**: 期限を過ぎたタスクに対して通知送信（1日1回の重複ガード）
- タスク毎の `notifyLevel` と `notifyTargets` 設定を尊重

## タブ構成: マイタスク + グループボード

### 概要

タスク管理画面は「マイタスク」と「グループボード（複数）」のタブ構成。旧スコープ（company/business/personal）は廃止し、`boardId` でグループボード紐付けを管理する。

- **マイタスク**: ボードに属さない自分専用タスク（`boardId = null` かつ `createdById = 自分`）。他のグループボードで担当者になっているタスクは含まない。
- **グループボード**: 用途自由（全社/事業/チーム/プロジェクト等）。Trelloのボード共有に近い仕組み。

### UIタブイメージ

```
[マイタスク] [📋 営業チーム] [📋 役員会] [📋 開発部] [+ グループボード]
```

- 先頭は常に「マイタスク」（固定）
- 参加中のグループボードがタブとして動的に表示（`TaskBoardMember.tabOrder` 順）
- `[+ グループボード]` ボタンでボード作成モーダル

### ボードタブ並び替え

- グループボードのタブ順序をドラッグ&ドロップで変更可能
- 並び順は `TaskBoardMember.tabOrder` にDB永続化（localStorageは使用しない）
- ユーザーごとに独立した並び順を保持
- 並び替えAPI: `PATCH /api/v1/task-boards/tab-order`（`{ orderedBoardIds: number[] }`）

### グループボード動作仕様

| 項目 | 仕様 |
|------|------|
| ボード作成 | 全ユーザー（admin / staff）が作成可能 |
| メンバー招待 | メンバー全員が他ユーザーを招待可能 |
| タスクの可視性 | ボード内タスクはメンバー全員が閲覧・編集可能 |
| 自分のタスクフィルター | 「自分のタスクを表示」トグルボタンで、自分が担当（`TaskAssignee`）のタスクのみに絞り込み表示 |
| ボード削除 | 作成者 + admin のみ |
| メンバー脱退 | 自分自身の脱退は可能、他メンバーの除外は作成者 + admin |

### ボード設定画面

- ボード名の編集
- メンバー一覧（ユーザー名 + ロール）
- メンバー招待（ユーザー検索 + 追加）
- ボード削除（作成者 / admin のみ）

### タスク作成時

- グループボードタブ表示中に新規作成すると、自動的にそのボードに紐付け
- マイタスクタブ表示中の新規作成では、ボード選択ドロップダウンで任意のボードを選択可能（未選択 = ボードなし）

## 画面構成

### メイン画面: `/tasks`

- タブ切替: マイタスク / グループボード（参加中ボード一覧）
- ビューモード切替: リスト / カンバン / カレンダー
- フィルター: 検索、ステータス（複数選択）、優先度（複数選択）、タグ（複数選択）、担当者
- 新規タスク作成ボタン

### リストビュー

- **divベースのCSS gridレイアウト**（tableタグ不使用）
  - `grid-cols-[28px_28px_84px_minmax(180px,1fr)_76px_64px_72px_92px_minmax(80px,160px)_64px_84px]`
  - ヘッダーはスクロール外に固定、ボディは `max-height: calc(100vh - 400px)` でスクロール
- 列: ドラッグハンドル / 展開 / No. / タスク名 / ステータス / 優先度 / 担当者 / 期限 / タグ / アーカイブ / 更新日
- **ステータス・優先度**: インラインドロップダウンで直接変更可能
- **アーカイブ**: 各行にチェックボックスで即時ON/OFF
- **ソート機能**:
  - デフォルトは手動順（`sortOrder:asc`）
  - 列ヘッダークリックで3段階切替: 昇順 → 降順 → 手動順に戻る
  - ドラッグハンドル列のヘッダー（GripVerticalアイコン）クリックで手動順に戻る
  - D&D並び替え後、手動順以外のソートの場合は自動で手動順（`sortOrder:asc`）に切替
- **一覧取得は常に `parentOnly=true`**（親タスクのみ取得）
  - サブタスクは展開時に `useTaskDetail(taskId)` で個別取得
- **親タスク展開でツリー表示**（└/├ + インデント + 青背景 + 縦ライン）
- 期限超過タスクは赤色表示
- デフォルトでアーカイブ済みタスク非表示（「アーカイブを表示」フィルターで表示）
- 担当者フィルター: ユーザー名検索ドロップダウン
- **ドラッグ&ドロップ**: dnd-kit SortableContext + PointerSensor（`distance: 8`）
  - 行の並び替え → reorder API で `sortOrder` 一括更新
  - DragOverlay でドラッグ中のプレビュー表示
  - ドラッグ開始時にサブタスク展開を閉じる
  - **DnDコールバック（`handleDragStart` / `handleDragEnd` / `handleDragCancel`）は `useCallback` で安定化**
- ページネーション

### カンバンビュー

- **カスタム列**: ユーザーが自由に列を作成・削除・名前変更・色設定
- **初回アクセス時デフォルト4列自動作成**: 未着手 / 進行中 / 保留 / 完了
- **列のD&D**: useSortable + horizontalListSortingStrategy + GripVerticalハンドル
- **カード内D&D**: useSortable + verticalListSortingStrategy（同一列内並び替え + 列間移動）
- **カスタム衝突検知**: 列ドラッグ中は列のみ検出（closestCenter）、カードドラッグ中は全て検出（closestCorners）
- **ローカルstate即時反映**: columnItemsRef + localColumnsRef でstale closure防止
- **カード内容**: タイトル、担当者、期限（「期限：」ラベル付き）、優先度（「優先度：」ラベル付き）、タグ
- **ステータス・優先度ドロップダウン**: カード上で直接変更可能
- **チェックリスト折りたたみ**: ChevronRight/Down + 直接チェック可能
- **サブタスク折りたたみ**: 階層表示（┣/┗）+ クリックで詳細遷移
- **列内タスク追加ボタン**: 各列に「タスクを追加」ボタン（`+` アイコン）で当該列に直接タスク作成
- **列追加ボタン**: ダッシュ枠ボタンで新規列追加
- **列メニュー**: 列名編集 / 列削除
- reorder API で columnId + sortOrder 一括更新

### カレンダービュー

- 月間グリッド（月曜始まりISO週、7列 x 週数）
- 期限日にタスクカードを表示（優先度で色分け）
- 月ナビゲーション（前月/翌月/今月リセットボタン）
- 親タスクのみ表示（parentTaskId !== null を除外）
- タスククリックで詳細パネル
- 外部日付ライブラリ不使用（全てローカルヘルパー）

### タスク詳細（サイドパネル）

タスク行クリックで右からスライドイン。

- **パンくずナビゲーション**（子タスク表示時: `親タスク > 子タスク`）
- タイトル（インライン編集可）
- ステータス / 優先度（セレクト）
- 担当者（複数担当者一覧表示、追加・削除可能）
- 期限（「なし」ボタンでクリア可能）
- URL（入力 + 別タブで開くボタン）
- タグ（**共通タグ/個人タグ チップ一覧** + 編集・削除 + 入力+サジェスト+新規作成）
- アーカイブ（チェックボックスで ON/OFF）
- 説明（テキストエリア）
- チェックリスト ☑（追加/チェック/削除、進捗バー表示）
- サブタスク 📋（一覧+追加、進捗バー表示、**クリックでサブタスク詳細に遷移**）

### 新規タスク作成（モーダル）

フィールド順序:
1. タスク名
2. ステータス + 優先度（横並び）
3. 担当者（マルチセレクト + フリーテキスト入力）
4. 期限（なし可）
5. URL
6. グループボード（ボード選択ドロップダウン、マイタスクタブ時のみ表示）
7. タグ
8. 説明
9. メモ（備考）
10. 通知設定

### ダッシュボードタスクウィジェット

会社ダッシュボード・事業別ダッシュボードの両方に表示。

- **サマリー**: todo / in_progress / 期限超過の件数
- **期限間近タスク**: 期限が近いタスク一覧（リンク付き）
- **期限超過タスク**: 期限を過ぎたタスク一覧（赤色表示）

※ 直近アクティビティセクションは不採用。

## レスポンシブ対応

全タスク管理画面はモバイルファーストで設計。CSS-onlyのTailwindブレークポイント（`sm:` 640px）で制御。

### カンバンビュー
- CSS `scroll-snap-type: x mandatory` で列を横スワイプ（Trello方式）
- モバイル: 各列が画面幅いっぱい（`min-w-[calc(100vw-2rem)]`）、スナップで1列ずつ切替
- デスクトップ: 従来通り `min-w-[260px] max-w-[300px]` の複数列表示
- D&Dは `PointerSensor`（distance: 8）で両デバイス共通

### カレンダービュー
- モバイル: セル高さ縮小（56px）、タスクはドット+件数表示、タップで日別一覧展開
- デスクトップ: セル高さ90px、タスクカード表示（最大3件 + 残数表示）

### 詳細パネル
- モバイル: 全幅（`w-full`）、ステータス/優先度が1列グリッド
- デスクトップ: `max-w-lg` 右パネル、2列グリッド
- iOS safe-area対応（`env(safe-area-inset-bottom)`）

### 作成モーダル
- モバイル: ボトムシート風（`items-end` + `rounded-t-xl`）、エッジtoエッジ全幅
- デスクトップ: 中央表示 `max-w-lg`
- ステータス/優先度: モバイル1列、デスクトップ2列

### メインページ（一覧）
- 検索: `w-full sm:w-64`
- フィルター: `flex-col sm:flex-row` で縦積み
- リスト: `overflow-x-auto` + `min-w-[900px]` で横スクロール
- 新規ボタン: `w-full sm:w-auto`

### タッチターゲット
- 全ボタン/リンク: モバイルで最小44px（`min-h-[44px] sm:min-h-0`）
- 削除/編集ボタン: モバイル常時表示（`flex sm:hidden sm:group-hover:flex`）
- カラーピッカー: モバイルで拡大（`h-6 w-6 sm:h-3.5 sm:w-3.5`）

## IME対応（日本語入力）

日本語入力（IME）の変換確定Enterでフォーム送信されないよう、全てのキーボードイベントハンドラーで `isComposing` チェックを実施。

- `onKeyDown` ハンドラーで `event.nativeEvent.isComposing === true` の場合は早期リターン
- 対象: タスク名入力、タグ入力、チェックリスト追加、検索フィールド等、Enterキーでアクション実行する全フォーム

## デプロイ時のDB更新

DBマイグレーションは `docker-entrypoint.sh` 内で `npx prisma migrate deploy` が自動実行されるため、新しいマイグレーションファイルを含むデプロイ時に手動でのマイグレーション作業は不要。ECSタスク起動時にエントリーポイントスクリプトが自動的に未適用のマイグレーションを検出・適用する。
