# 管理システム 簡易マニュアル

---

## はじめに

本マニュアルは、管理システムの導入から日常業務での活用方法を解説するものです。
**管理者（admin）** と **一般ユーザー（staff）** の2つの視点で構成しています。

### 本システムでできること

| 機能 | 概要 |
|------|------|
| 顧客管理 | 顧客情報の一元管理（連絡先・銀行口座含む） |
| 代理店管理 | 代理店の階層構造・事業別の関係管理 |
| 契約マスタ | 営業案件の進捗・ステータス・カスタムフィールド管理 |
| ムーブメント管理 | 案件ごとの営業プロセス（ステップ）の進捗管理 |
| ダッシュボード | KPI・パイプライン・売上推移・代理店ランキングの可視化 |
| タスク管理 | マイタスク・グループボードでのタスク管理（カンバン・カレンダー） |
| プッシュ通知 | スマホ・PCへのリアルタイムプッシュ通知（iPhone Safari / Android Chrome / PC対応） |
| AIアシスタント | 営業データに対する自然言語での問い合わせ |
| 売上目標・レポート | 月次売上目標の設定と実績レポート |
| 資料共有 | 代理店向けの資料・請求書の配信管理 |
| QA / 問い合わせ | ナレッジベースとサポート窓口 |

### ユーザーの種類

| ロール | 説明 | アクセス範囲 |
|--------|------|-------------|
| admin（管理者） | システム全体の設定・管理が可能 | 全機能 |
| staff（社内スタッフ） | 日常の営業業務を行う | 管理者設定画面を除く全機能 |
| partner_admin（代理店管理者） | 代理店ポータルの管理 | 代理店ポータル + スタッフ管理 |
| partner_staff（代理店スタッフ） | 代理店ポータルの閲覧 | 代理店ポータル（閲覧のみ） |

---

# 第1部：管理者（admin）向けマニュアル

> **対象読者**: システム管理者。初期設定からマスタ整備、日常の管理業務まで。

---

## 1. システム導入の推奨フロー

新しくシステムを使い始める場合、以下の順番で設定するとスムーズです。

```
Step 1  事業マスタの作成
   ↓
Step 2  事業ごとの詳細設定（カスタムフィールド・ステータス定義・KPI等）
   ↓
Step 3  ユーザーの登録（社内スタッフ + 代理店ユーザー）
   ↓
Step 4  顧客マスタの登録（CSV一括 or 個別登録）
   ↓
Step 5  代理店マスタの登録（階層構造を含む）
   ↓
Step 6  売上目標の設定
   ↓
Step 7  必要に応じてAIアシスタント・通知等のシステム設定
```

以降、各ステップを詳しく説明します。

---

## 2. Step 1：事業マスタの作成

**画面**: サイドバー「事業マスタ」→「新規作成」

事業マスタはシステムの中核です。契約マスタ・ダッシュボード・ムーブメント等、多くの機能が事業に紐づきます。

### 操作手順
1. サイドバーの「事業マスタ」をクリック
2. 「新規作成」ボタンをクリック
3. 以下を入力して保存
   - **事業コード**: 英数字の一意コード（例: `SALES01`）
   - **事業名**: 表示名（例: 「法人営業事業」）

### ポイント
- 事業コードは後から変更できません。慎重に設定してください
- 事業を作成すると、サイドバーの事業セレクターに表示されます
- 契約マスタやムーブメントは事業を選択しないと表示されません

---

## 3. Step 2：事業ごとの詳細設定

**画面**: 事業マスタ詳細画面の各タブ

事業を作成したら、以下の4つの設定を行います。

### 3-1. 案件フィールド設定（カスタムフィールド）

**タブ**: 「案件フィールド設定」

案件（契約マスタ）で管理する項目を自由に定義できます。

| フィールド種別 | 用途例 |
|--------------|--------|
| テキスト | 備考、ニーズ、特記事項 |
| 数値 | 売上金額、台数、面積 |
| セレクト（プルダウン） | 商品種別、ランク |
| チェックボックス | 承認済み、確認済み |
| 日付 | 契約日、納品予定日 |
| 月（YYYY-MM） | 対象月、開始月 |
| URL | 関連資料リンク |
| テキストエリア | 長文のメモ |
| 計算式（formula） | 他フィールドの演算結果（例: `単価 * 数量`） |

**操作手順**:
1. 「フィールド追加」をクリック
2. フィールド名・種類・必須/任意を設定
3. 必要に応じて「代理店に公開」「絞り込み対象」のチェックを設定
4. ドラッグ&ドロップで表示順を変更
5. 保存

**Tips**:
- 「代理店に公開」にチェックすると、代理店ポータルでもそのフィールドが表示されます
- 「絞り込み対象」にチェックすると、一覧画面のフィルターに追加されます
- 計算式フィールドでは `{フィールドキー}` で他のフィールドを参照できます

### 3-2. 営業ステータス定義

**タブ**: 「営業ステータス」

案件の進捗を管理するステータスを定義します。

**操作手順**:
1. 「ステータス追加」をクリック
2. ステータス名（例: 提案、検討中、受注見込み、受注、失注）を入力
3. 表示色を選択
4. 必要に応じて以下を設定:
   - **最終ステータス**: 完了扱い（デフォルトで一覧から非表示）
   - **失注ステータス**: 失注扱い（デフォルトで一覧から非表示）
   - **ソート順**: ステータスの並び順

### 3-3. KPI定義

**タブ**: 「KPI設定」

ダッシュボードで表示するKPIを定義します。

**操作手順**:
1. 「KPI追加」をクリック
2. KPIラベル（例: 売上金額、受注台数）を入力
3. 集計対象のフィールドを選択（カスタムフィールドの数値型）
4. 対象とする営業ステータスを選択（複数選択可）
5. 単位フォーマットを設定（例: `#円` → `1,000円`、`#台` → `100台`）

### 3-4. ムーブメント定義

**タブ**: 「ムーブメント定義」

案件の営業プロセスを段階的に管理するためのテンプレートを作成します。

**操作手順**:
1. 「ステップ追加」でプロセスの各段階を定義
2. ステップ名を入力（例: 初回訪問、提案書提出、見積提出、契約締結）
3. 必要に応じてステップに案件フィールドを紐付け（そのステップで入力/表示するフィールド）
4. ドラッグ&ドロップで順序変更
5. 保存

**Tips**:
- ムーブメント定義を更新すると、既存の案件にも自動で同期されます
- 手動同期ボタンで任意のタイミングでも同期可能です

---

## 4. Step 3：ユーザー登録

**画面**: ユーザーメニュー → 「ユーザー管理」（admin のみ）

### 社内ユーザーの登録
1. 「ユーザー追加」をクリック
2. 氏名・メールアドレスを入力
3. ロールを選択（admin or staff）
4. パスワードを設定（「自動生成」ボタンあり）
5. アクセスさせる事業を選択（複数可）
6. 保存

### 代理店ユーザーの登録
1. 「ユーザー追加」をクリック
2. 氏名・メールアドレスを入力
3. ロールを選択（partner_admin or partner_staff）
4. 所属する代理店を選択（検索コンボボックスで絞り込み可）
5. パスワードを設定
6. アクセスさせる事業を選択
7. 保存

### ポイント
- 代理店管理者（partner_admin）は、自社の partner_staff をポータルから追加・管理できます
- ユーザー一覧は「階層表示」「フラット表示」の切替が可能です

---

## 5. Step 4：顧客マスタの登録

**画面**: サイドバー「顧客マスタ」

### 個別登録
1. 「新規作成」をクリック
2. 顧客コード・顧客名・住所・連絡先等を入力
3. 保存
4. 詳細画面で連絡先・銀行口座を追加（タブから操作）

### CSV一括登録
1. 「CSVインポート」をクリック
2. テンプレートをダウンロード（「テンプレートDL」ボタン）
3. テンプレートにデータを入力
4. CSVファイルをアップロード → **ドライラン実行**
5. エラーがあればエラーレポートをダウンロードして修正
6. エラーがなければ「確定」で登録

### ポイント
- ドライランでデータの妥当性を事前確認できます（実データには影響しません）
- 顧客コードは一意です。重複するとエラーになります

---

## 6. Step 5：代理店マスタの登録

**画面**: サイドバー「代理店マスタ」

### 階層構造について
- 代理店はN次の親子階層を持てます（例: 1次代理店 → 2次代理店 → 3次代理店）
- マスタ階層（全体共通）と事業別階層（事業ごとに異なる親子関係）の2種類があります

### 操作手順
1. 「新規作成」をクリック
2. 代理店コード・代理店名を入力
3. 親代理店を選択 → 階層（Tier）が自動決定されます
4. 保存
5. 詳細画面で連絡先・銀行口座・事業リンクを設定

### ポイント
- 事業別の親子関係は「事業リンク」タブで設定します
- 一覧表示では「スコープ切替」で「マスタ階層」「事業別階層」を切り替えて確認できます

---

## 7. Step 6：売上目標の設定

**画面**: サイドバー「売上目標」（事業選択が必要）

### 操作手順
1. サイドバーの事業セレクターで対象事業を選択
2. 「売上目標」をクリック
3. 対象年度を選択
4. 月別に目標値を入力
5. 保存

### ポイント
- ダッシュボードの「達成率」はここで設定した目標値に対する実績比です
- KPIごとに目標を設定できます

---

## 8. Step 7：システム設定

**画面**: ユーザーメニュー → 「システム設定」（admin のみ）

### AIアシスタント設定
1. OpenAI APIキーを入力（暗号化して保存されます）
2. モデル選択: 自動（推奨）/ GPT-4o / GPT-4o-mini
3. 保存

### プッシュ通知設定
プッシュ通知は環境変数（VAPID鍵）の設定のみで有効になります。管理画面での追加設定は不要です。
ユーザーは通知ベルのポップオーバーから自分で有効化します。

### 資料共有の管理
**画面**: 事業マスタ詳細 → 「資料」タブ / 「明細」タブ

1. ファイルをアップロード
2. ドラッグ&ドロップで表示順を変更
3. 「通知送信」ボタンで代理店に通知（再送も可能）

### お知らせ管理
**画面**: ユーザーメニュー → 「お知らせ管理」

1. 「新規作成」をクリック
2. タイトル・本文・優先度・対象（全体 or 特定事業）を設定
3. 公開日を設定して保存

---

## 9. 管理者の日常業務

### 日次
- ダッシュボードでKPI・パイプラインを確認
- 新規の問い合わせ対応
- タスクの進捗確認

### 週次
- 代理店ランキングの確認
- ムーブメントの進捗確認（マトリクス・ガントチャート）

### 月次
- 月次レポートの確認・配信
- 売上目標の実績対比
- 必要に応じて資料共有の更新・通知送信

# 第2部：一般ユーザー（staff）向けマニュアル

> **対象読者**: 日常の営業業務でシステムを利用するスタッフ

---

## 1. ログインと画面構成

### ログイン
1. ブラウザでシステムURLにアクセス
2. メールアドレスとパスワードを入力
3. 「ログイン」をクリック

### 画面構成

```
┌─────────────────────────────────────────┐
│ ヘッダー [AI] [QA] [問合せ] [ユーザーメニュー]  │
├──────┬──────────────────────────────────┤
│      │                                  │
│ サイド │         メインコンテンツ           │
│ バー   │                                  │
│      │                                  │
├──────┴──────────────────────────────────┤
```

**サイドバー**:
- ダッシュボード / タスク管理 / 契約マスタ / 案件ムーブメント / 事業マスタ / 顧客マスタ / 代理店マスタ / 月次レポート

**ヘッダー右上アイコン**:
- AIアシスタント / QA・ナレッジ / 問い合わせ / リマインダー🔔 / 通知🔔

**通知ベル🔔**: 未読件数がバッジ表示。クリックで通知ポップオーバーを開き、プッシュ通知のトグル切替・通知の既読/削除もここから行います。

**ユーザーメニュー**（右上プロフィールアイコン）:
- お知らせ管理 / 文字サイズ変更 / ログアウト

**Tips**: 事業セレクターで事業を選択すると、契約マスタやムーブメントが表示されます。

---

## 2. 日常業務の推奨フロー

```
朝   → ダッシュボードでKPI確認 + タスク確認
午前 → 案件ムーブメントで進捗更新
午後 → 契約マスタの更新（新規案件登録・ステータス変更）
夕方 → タスクの完了報告 + 翌日タスク確認
```

## 3. ダッシュボード

**画面**: サイドバー「ダッシュボード」

ログイン後に最初に表示される画面です。

### 見方

| セクション | 表示内容 |
|-----------|---------|
| KPIサマリー | 実績値・目標達成率・前月比 |
| パイプライン | 営業ステータス別の件数・金額 |
| 売上推移グラフ | 年度内の月別推移（実績 + 目標ライン） |
| 代理店ランキング | 代理店別の売上順位 |
| タスク | 期限間近・期限超過のタスク |

### 操作
- **期間フィルター**: 上部で「単月 / 期間指定 / 全期間」を切替
- **単月ナビ**: 単月モード時に `<` `>` で前月/翌月を移動、「当月に戻す」で今月に戻る
- **KPI切替**: 複数のKPIが定義されている場合、セレクターで切り替え
- **事業切替**: ヘッダーの事業セレクターで全社/事業別を切替

---

## 4. 契約マスタ（案件管理）

**画面**: サイドバー「契約マスタ」（事業選択が必要）

### 案件の新規登録
1. 「新規作成」をクリック
2. 顧客・代理店をプルダウンで選択
3. 営業ステータスを選択
4. 受注予定月を設定
5. カスタムフィールド（金額、ニーズ等）を入力
6. 必要に応じて「ポータルに表示」チェックを設定（デフォルトON。OFFにすると代理店ポータルに非表示）
7. 保存 → 案件番号が自動採番されます

### 一覧画面の使い方

**フィルター**:
- 営業ステータスで絞り込み（複数選択可）
- 受注予定月で絞り込み（単月 / 範囲指定 / 全件）
- テキスト検索（顧客名・代理店名・案件番号）
- カスタムフィールドのフィルター（管理者が「絞り込み対象」に設定したもの）

**列設定**:
- ヘッダー右の歯車アイコンで列の表示/非表示を変更
- 列をドラッグ&ドロップで並び替え
- 「すべて表示」「すべて解除」ボタンで一括操作
- 設定した列はブラウザに保存されます

**ビューの保存**:
- フィルターや列設定を「ビュー」として保存可能
- タブで保存済みビューを素早く切替
- 「チームに共有」で他のメンバーと共有可能

**CSV操作**:
- エクスポート: 現在の表示条件でCSV出力
- インポート: テンプレートDL → データ入力 → アップロード → ドライラン確認 → 確定

### ポイント
- 「最終」「失注」ステータスの案件はデフォルトで非表示です。フィルターで該当ステータスを選択すると表示されます
- ピン機能: よく確認する案件を上部に固定できます

---

## 5. 案件ムーブメント

**画面**: サイドバー「案件ムーブメント」（事業選択が必要）

案件ごとの営業プロセスの進捗をマトリクス形式で管理します。

### 画面の見方
- **縦軸**: ムーブメント定義のステップ（初回訪問、提案書提出 等）
- **横軸**: 各案件（顧客名・代理店名・案件番号が表示）
- **セル**: 各ステップの実施状況

### 操作
1. セルをクリックするとモーダルが開く
2. 日付・コメント・ステータスを入力
3. ステップに紐付けられたフィールド（ニーズ等）がある場合、ここで編集可能
4. 保存

### フィルター
- 営業ステータスで絞り込み
- 受注予定月で絞り込み
- テキスト検索（顧客名・案件番号）
- カスタムフィールドフィルター（折りたたみパネル内）

### ポイント
- 案件情報セルには「ニーズ」等の重要情報が表示されます
- ヘッダーの「営業ステータス」「受注予定月」列はクリックでソート可能（昇順→降順→解除）
- 案件情報・営業ステータス列はスクロール時も固定表示されます

---

## 6. ガントチャート

**画面**: サイドバー「ガントチャート」（事業選択が必要）

ムーブメントの進捗をタイムライン形式で確認できます。

### 操作
- **表示切替**: 週ビュー / 月ビュー
- **期間移動**: 前月 `<` / 翌月 `>` ボタン
- **今日の線**: 赤い縦線で現在日を表示

---

## 7. タスク管理

**画面**: サイドバー「タスク管理」

### マイタスクとグループボード
- **マイタスク**: 自分だけのタスク管理（他のボードの担当タスクとは独立）
- **グループボード**: チームで共有するタスクボード（全社・事業・チーム等、用途自由）

### ビュー切替
- **リスト**: 一覧表示（フィルター・ソート対応）
- **カンバン**: ステータス列ごとのカード表示（ドラッグ&ドロップ対応）
- **カレンダー**: 期限日ベースの月間カレンダー

### タスクの作成
1. 「タスク追加」をクリック（カンバンでは各列の「+」ボタン）
2. タイトルを入力
3. 必要に応じて設定:
   - 担当者（複数選択可、外部担当者はフリーテキストで追加）
   - 優先度（緊急・高・中・低）
   - 期限日
   - タグ
   - サブタスク・チェックリスト
   - 添付ファイル（ドラッグ&ドロップ or Ctrl+V で貼り付け可能、10MB/10ファイルまで）
4. 保存

### タスクの移動・コピー

タスク詳細パネルの右上にある `⋯`（アクションメニュー）から操作できます。

**ボードに移動**:
1. タスク詳細パネルの `⋯` をクリック
2. 「ボードに移動」を選択
3. 移動先を選択（マイタスク or グループボード）
4. タスクが移動先のボードに移動します

**タスクをコピー**:
1. タスク詳細パネルの `⋯` をクリック
2. 「タスクをコピー」を選択
3. タイトル末尾に「（コピー）」がついた複製タスクが作成されます

**ポイント**:
- コピーでは担当者・タグ・チェックリスト等が引き継がれます
- コピーではステータスは「未着手」にリセットされます
- サブタスクと添付ファイルはコピーされません
- ボード移動時、カンバン列はリセットされます（ボード間で列は共有されないため）

### カンバンの操作
- カードをドラッグ&ドロップで別の列に移動すると、タスクのステータスも自動的に更新されます
- 列自体もドラッグ&ドロップで順序変更可能
- 「自分のタスクを表示」トグルで自分の担当タスクのみ表示

## 8. プッシュ通知

### プッシュ通知とは

ブラウザやアプリを閉じていても、スマホやPCに通知が届く機能です。
以下のイベントでプッシュ通知が送信されます：

| イベント | 説明 |
|---------|------|
| 営業ステータス変更 | 案件のステータスが変更された時 |
| タスクアサイン | タスクに担当者として割り当てられた時 |
| タスク完了 | 担当タスクが完了した時 |
| タスク期限超過 | 担当タスクの期限が過ぎた時（日次cronジョブ） |
| 資料共有通知 | 管理者が資料・明細の通知を送信した時 |
| システム通知 | その他のシステム通知 |

### プッシュ通知の有効化 / 無効化

通知ポップオーバー上部の**トグルスイッチ**で切り替えます。

1. ヘッダー右上のベルアイコン🔔をクリック
2. 「プッシュ通知」の横にあるトグルスイッチをONにする
3. ブラウザの許可ダイアログで「許可」を選択
4. 以後、プッシュ通知が届くようになります

無効にする場合は、同じトグルスイッチをOFFにしてください。

### デバイス別の対応状況

| デバイス | ブラウザ | 条件 |
|---------|---------|------|
| **PC** | Chrome / Edge / Firefox | ブラウザで有効化するだけでOK |
| **Android** | Chrome | ブラウザで有効化するだけでOK（ホーム追加は任意） |
| **iPhone** | **Safariのみ** | **ホーム画面に追加が必須**（下記手順参照） |

> ⚠️ **iPhoneの注意点**: iPhone では Safari 以外のブラウザ（Chrome等）ではプッシュ通知が届きません。必ず Safari を使用してください。

### iPhoneでのホーム画面追加手順

1. **Safari** で管理システムのURLを開く
2. 画面下部の **共有ボタン**（□に↑のアイコン）をタップ
3. 「**ホーム画面に追加**」をタップ
4. 名前を確認して「追加」をタップ
5. **ホーム画面から** アプリアイコンをタップして起動
6. ベルアイコン → トグルスイッチをON → 「許可」

> iOS 16.4以降が必要です。

### 通知の管理

通知はベルアイコン🔔のポップオーバーから管理できます。

| 操作 | 方法 |
|------|------|
| 既読にする | 通知をクリック（関連画面に遷移） |
| すべて既読 | ヘッダーの「すべて既読」ボタン |
| 個別削除 | 通知にマウスを合わせて表示される**×ボタン**をクリック |
| すべて削除 | ヘッダーの「すべて削除」ボタン |

**ポイント**:
- 通知はDBに蓄積されます。不要な通知は削除してください
- 既読にしても通知は残ります（削除は別操作）
- 代理店ポータルでも同じ通知ベルが表示されます

### トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| トグルスイッチが表示されない | ブラウザがプッシュ通知に対応していない可能性があります |
| 「ブロックされています」と表示 | ブラウザの設定で通知を許可してください |
| iPhoneで通知が届かない | Safariでホーム画面に追加してから起動しているか確認してください |
| 複数デバイスで通知が届かない | 各デバイスで個別に「有効にする」操作が必要です |

## 9. AIアシスタント

**画面**: ヘッダーのAIアイコン → AIアシスタント

営業データに自然言語で質問できる機能です。

### 使い方
1. ヘッダーのAIアイコンをクリック
2. チャット欄に質問を入力（Shift+Enter で送信、Enter で改行）
3. 回答が表示される（表形式の回答にも対応）

### 質問例
- 「今月の売上実績を教えて」
- 「パイプラインの状況は？」
- 「代理店Aの案件一覧を見せて」
- 「前月比で受注金額はどう変わった？」
- 「受注見込みの案件を金額順で一覧して」

### ポイント
- サイドバーで選択中の事業に基づいてデータを取得します
- 会話履歴は保存されるため、過去の質問を振り返ることができます
- 管理者がAPIキーを設定していない場合は利用できません

## 10. テーブル共通操作

システム内の多くの一覧画面で使える共通操作です。

### 列設定
1. テーブルヘッダーの歯車アイコンをクリック
2. 表示/非表示をチェックボックスで切替
3. ドラッグ&ドロップで列の並びを変更
4. 「すべて表示」「すべて解除」で一括操作

### テーブルヘッダー
- テーブルヘッダーはスクロール時も固定表示されます
- 列名をクリックするとソート（昇順→降順→解除）。複数列に対してソートでき、優先度が番号で表示されます

### ビュー管理
- 「ビューを保存」: 現在の列設定・フィルターを名前付きで保存
- タブで保存済みビューを素早く切替
- 「チームに共有」にチェックすると他ユーザーも利用可能（読み取り専用）
- 共有ビューは「自分のビューにコピー」でカスタマイズ可能

### 表示件数
- テーブル下部のセレクターで表示件数を変更
- 設定はビューごとに保存されます

# 第3部：代理店ポータルユーザー向け（参考）

> 代理店ユーザー（partner_admin / partner_staff）は専用のポータル画面を使用します。

### アクセスできる画面
- **ダッシュボード**: 自社の売上・KPI・ランキング
- **契約一覧**: 自社に関連する案件の閲覧
- **案件ムーブメント**: 自社案件の進捗確認（閲覧のみ）
- **月次レポート**: 自社の月次実績
- **QA・ナレッジ**: 公開されたFAQの閲覧
- **問い合わせ**: サポート窓口への問い合わせ

### partner_admin のみの機能
- **スタッフ管理**: 自社のポータルユーザーの追加・編集・削除

### ポイント
- ポータルでは管理者が「代理店に公開」に設定したフィールドのみ表示されます
- 管理者が「ポータルに表示」をOFFにした案件はポータルに表示されません
- ムーブメントのセル編集はできません（閲覧のみ）
- ダッシュボードでは売上目標のラインは非表示です
